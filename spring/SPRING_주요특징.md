- DI란?
Dependency Injection. 의존성 주입을 뜻하는 말로, 객체 간의 의존성을 프레임워크가 주입해주는 방식.

DI 방식은 3가지이다.

1. 필드 주입, Autowired어노테이션을 사용하여 스프링 처음 개발을 시작했을때 사용했던 방식이며, 작성이 간단하다는 장점이 있지만 그것 외에는 불변성 보장 불가능, 테스트 어려움(mock 객체 주입 어려움) 등의 단점이 존재한다.

2. 생성자 주입, 현재 스프링에서 가장 권장하는 의존성 주입 방식이다. 클래스 생성시 생성자를 통해 객체의 의존성을 주입하기 때문에 final 키워드와 함께 사용하여 불변성을 보장할 수 있으며 순환 참조를 방지할 수 있고 테스트 또한 용이하다. - 현재는 개인적으로 이 방식을 사용.

3. 수정자주입, set 메소드에 autowired 어노테이션을 사용하는 방식이다. 주입받는 객체를 선택적으로 사용하는 경우에 사용하며, 개인적으로 아직 이 방법은 프로젝트시에 실제로 사용한 적이 없는 방식이다.

- IOC란?
객체의 생명주기 관리를 프레임워크에게 위임하는 것. 제어의 역전.
EX) @Component, @Autowired 등

- AOP란?
관점 지향 프로그래밍. 공통 로직을 비즈니스 로직에서 분리하여 유지보수성과 가독성을 높임.
로깅, 트랜잭션 등에서 사용.
